Aprendiendo gitHub

Comandos Git
  1. git clone
    Permite clonar un proyecto de nuestro repositorio en la web
    Pasos:
    • Abrir terminal
    • Digitar: git clone "https://github.com/Zuv0x/hello.git"
      1. Crear un archivo
        2. touch hello.html
      2. Listar archivos
        3. ls
  2. git add
    3. Permite agregar un archivo para que este en seguimiento
    Pasos:
    • git add hello.html
  3. git commit -m
    4. Permite capturar el documento y agregar un mensaje
    Pasos:
    • git commit -m "Comandos git"
    • git commit -am "Guarda todos los archivos modificados y agrega un mensaje"
  4. git status
    5. Muestra el estado de los archivos en mi directorio de trabajo
    Pasos:
    • git status
  5. git push
    6. Enviará nuestro directorio local y actualizara el repositorio de gitHub.
    Pasos:
    • git push
  6. git pull
    7. Tomará el repositorio en gitHub y lo reemplazara por el local
    Pasos:
    • git pull
    • En caso de conflictos aparecerá "CONFLICT: merge conflict"
  7. git log
    8. Muestra el historico de commit
    Pasos:
    • git log
      • Para observar todos desplazamos con las teclas de dirección en la pantalla
    • git log --oneline
      • Muestra lista de una linea
    • git log -X
      • Muestra los X(número) commit últimos
    • git log -2 -p
      • Muestra el commit 2 y el detalle del codigo únicamente de ese commit
    • git show "id"
      • Muestra el commit y su detelle. El id se obtiene del git log --oneline
      referencia: https://desarrolloweb.com/articulos/git-log.html
  8. git reset
    9. Retorna a un punto anterior del repositorio
    Pasos:
    • git reset --hard "commit"
      • Para retornar a una confirmación específica
    • git reset --hard "origin/master"
      • Para retornar a la version del repositorio gitHub
  9. git branch
    10. Permite trabajar con ramas
    Pasos:
    • git branch
      • Muestra las ramas de mi repositorio
    • git checkout -b name
      • Crea una rama nueva de nombre "name"
      Activa la nueva rama para su uso
    • git checkout master
      • Retorna a la rama principal o la que querramos llamar
    • git merge "rama2"
      • Fusiona las ramas elegidas
  10. Button Fork
    11. Bifurcar repositorio: Hacer una copia propia a nuestro repositorio
    Ejemplo: https://github.com/twbs/bootstrap
